# Ozon Dashboard

Дашборд для продавца Ozon: себестоимость (диффузоры/саше), прогнозы, продажи с полем «фактически получено», графики с переключаемой легендой, выгрузка в Excel.

## Запуск (Mac)

1. Скопируйте `.env.example` в `.env` и укажите:
   - `OZON_CLIENT_ID` — ID из личного кабинета Ozon (Настройки → API)
   - `OZON_API_KEY` — ключ API

2. Запуск одним кликом:
   - Двойной клик по **`start.command`** (откроется терминал и поднимется сервер)
   - Или в терминале: `./start.sh`

3. Откройте в браузере: **http://localhost:3000**

## Деплой на Railway

1. Подключите репозиторий к [Railway](https://railway.app), выберите этот проект.
2. В настройках сервиса добавьте переменные окружения: `OZON_CLIENT_ID`, `OZON_API_KEY` (значения из .env).
3. **Чтобы расходники, пресеты и данные не пропадали после передеплоя** — добавьте том (Volume): в Railway в настройках сервиса создайте Volume и примонтируйте его, например, в `/data`. Затем добавьте переменную окружения **`DATA_DIR=/data`**. Все JSON-файлы (продажи, расходники, пресеты и т.д.) будут храниться на томе. **Без тома и DATA_DIR при каждом обновлении сервера данные обнуляются.**
4. В разделе **Settings → Networking** нажмите **Generate Domain** — Railway выдаст публичный URL.
5. Сервис запускается командой из `Procfile`: `web: node server.js`. Порт берётся из `process.env.PORT`.

## Разделы

- **Продажи и доходы** — карточки (Получено, Чистая прибыль, Расходы, Реклама), график с выбором периода и переключателями рядов в легенде, структура расходов, маржинальность, таблица продаж с полем «Сколько фактически отправил Озон», кнопки «Обновить продажи» и «Выгрузить в Excel».
- **Себестоимость** — стоимость товаров на складе, расходники (с типом: саше/диффузоры/оба), привязка товаров к типу, таблица себестоимости и прогноз «хватит на N дней».
- **Товары** — вкладки: Остатки (+10 к выбранным), Цены, Описания (переносы строк → `<br>`).

## Зависимости

- Node.js 18+
- Пакеты: `express`, `dotenv`, `xlsx` (устанавливаются через `npm install`).
